# 雪球特别关注 新帖提醒（Chrome 扩展 / Manifest V3）

监视雪球「特别关注」分组里的博主，他们一发新帖，就在**本机屏幕**弹出醒目的系统通知（高优先级、不自动消失、点一下直达原帖）+ **右侧磁吸常驻弹窗**（按人合并、已读消隐、支持全部一键已读）。

借用**已登录 Chrome 的免费登录态**直接调雪球半公开 API，**不需要**提取 Cookie、不需要常驻服务。可选地把新帖**推送进你的个人微信**（企业微信自建应用，见下方「④ 企业微信推送」）。

> **为什么不做独立 App / 微信小程序？** 手机沙盒隔离读不了另一个 App 的数据；小程序无后台定时能力、无系统通知；独立 App iOS 保活是硬坑且需开发者账号。当前方案：电脑前用桌面弹窗（最醒目），离开电脑用企微推到手机微信（置顶会话=最接近弹窗的原生形态）。

---

## 安装（加载已解压扩展）

1. 打开 Chrome，地址栏输入 `chrome://extensions`
2. 右上角打开 **开发者模式（Developer mode）**
3. 点 **加载已解压的扩展程序（Load unpacked）**
4. 选择本目录（`xueqiu-chrome-extension/`）
5. 浏览器右上角会出现扩展图标，点开即弹出面板

> 因为是本地 unpacked 扩展，Chrome 每次启动会自动启用，无需重新加载。若要更新代码，回到 `chrome://extensions` 点该扩展的「刷新」图标即可。

## 使用

- **前提**：Chrome 已登录雪球（`xueqiu.com`），且你确实有「特别关注」分组。
- 扩展安装后会**立即跑一次检查**——这次只记录当前最新帖、不发通知（避免一上来弹一堆历史帖）。
- 之后按设定间隔（默认 2 分钟）轮询。发现比记录更新的帖就弹通知。
- 点通知或通知上的「打开原帖」按钮 → 在新标签页打开雪球原帖。
- 点扩展图标可看：监控人数、上次运行时间、每人抓取条数、最近新帖列表，以及「立即检查一次」。若「上次检查出错」标红，按提示排查。
- 想先验证提醒链路通不通？打开**设置页**点「📢 发送测试提醒」，会立刻弹一条测试通知 + 小窗（声音需在设置里开启）。

### 设置（点面板底部「⚙️ 设置」）
1. **检测登录**：点一下，扩展自动借已登录的雪球标签页读取登录态，显示「✅ 已登录 — 用户名：xxx」。无需 DevTools、无需手动复制 Cookie。
2. **选择要监控的博主**：点「📋 读取特别关注分组」，自动拉出你的特别关注成员列表，勾选你想监控的人即可（勾选结果自动写入监控名单）。
3. **轮询间隔**：1–60 分钟，默认 2。雪球未登录约 15 次/分钟限流，太短易触发 429。
4. **兜底博主名单**：填雪球 `user_id`（逗号或换行分隔）。当「特别关注」分组取不到时回退到此名单；留空则只依赖分组。

## 文件说明

| 文件 | 作用 |
|---|---|
| `manifest.json` | MV3 清单，声明 alarms / notifications / storage / offscreen / windows / **tabs / scripting** 权限、content_scripts（注入 xueqiu.com）与主机权限 |
| `background.js` | service worker：定时轮询、调雪球 API、比对新帖 → 系统通知 + 提示音 + 弹窗；含 `diagnose()` 诊断 |
| `content.js` | 注入到 xueqiu.com 页面的内容脚本：在雪球同源上下文里代发 `fetch`，自动携带全部 Cookie（含 httpOnly） |
| `offscreen.html` / `offscreen.js` | 离屏文档：后台用 Web Audio 播「滴-滴」提示音（无音频文件），被拦截时自动改系统朗读 |
| `alert.html` / `alert.js` | 新帖时弹出的小窗口：按博主合并未读卡片、点击打开原帖+标已读、**全部已读按钮**、3 分钟未读重弹、常驻不自动关闭（全读完才关） |
| `popup.html` / `popup.js` | 点击图标弹出的状态面板 |
| `options.html` / `options.js` | 设置页（检测登录、勾选博主、轮询间隔、声音开关、**一键诊断**） |
| `icon16.png` / `icon48.png` / `icon128.png` | 扩展图标（渐变雪球 + 立体白雪晶） |

## 原理 & 已验证接口

扩展**借你已登录的 Chrome 里的雪球标签页**自动调 API，三层路径按"最稳优先"自动选用，你一般什么都不用做：

1. **主路径（scripting 随用随注入）**：用 `chrome.scripting.executeScript` 在已打开的雪球标签页里**临时注入 fetch**——同站同源、浏览器自动带全部 Cookie（含 httpOnly 的 `xq_a_token`），**不需要你去 F5 刷新任何页面**，随用随注入，最稳。
2. **次路径（content script）**：若标签页已加载过扩展、注入了 `content.js`，直接 `sendMessage` 代发，比 ① 少一次注入开销、更快；标签页没刷新导致未注入时会报 "Receiving end does not exist"，此时自动回退 ①，**无影响、可忽略**。
3. **兜底（手动 Cookie）**：极端情况下（没有任何雪球标签页 / 注入被拦），可在设置页「高级」里手动粘贴 Cookie 救急。

> **为什么不再强依赖手动粘 Cookie**：实测借标签页注入（`scripting`）在已登录 Chrome 里稳定带登录态（诊断 ②b 已实锤返回真实「特别关注」分组），比手动复制 Cookie 更省事、更不易出错。手动 Cookie 仅作最后兜底。

**注意**：Chrome 必须保持打开且雪球已登录（保持至少一个雪球标签页），否则收不到提醒。

- 取特别关注分组：`GET /friendships/groups.json` → **登录后返回顶层数组 `[{...分组...}]`**（未登录返回 `{error_code:400016}`）→ 找 name 含「特别关注」/`special` 的分组 → 成员 `GET /friendships/groups/members.json?gid=<id>`（部分账号 `groups[].users` 直接带）
- 取用户时间线：`GET /v4/statuses/user_timeline.json?user_id=<id>&page=1`（v4 是当前活接口，旧 `/statuses/user_timeline.json` 在部分账号下返回 400 仅作兜底）。**已修复字段名兼容**：不同版本返回 `statuses` / `list` / `items` / `data.*` 都能正确解析，避免"有人发帖却判为空、不提醒"。
- 去重：`chrome.storage.local` 记录每个用户已见过的最大帖 id，只推送更大的新帖
- 偶尔遇到 `acw_sc__v2` JS 加密导致接口异常时，手动开一下雪球网页即可刷新 Cookie

> 这些接口已对照你之前成功的 JS 版仓库（`github.com/JohnWish1590/xueqiu`）核实可用。

## 三层提示 + 企微推送（电脑前与离开电脑都醒目）

新帖触发时同时做多件事：

1. **系统通知**（默认开）：Windows 右下角 toast，`priority:2` + `requireInteraction:true`，即便 Chrome 最小化也会在桌面出现、不自动消失。
2. **声音提示**（默认关，设置里勾选开启）：用 Offscreen Document + Web Audio 播「滴-滴」声，**Chrome 最小化也响**；若被浏览器自动播放策略拦住，自动改用系统朗读「雪球特别关注，有新帖」。首次需在设置页勾选一次以授权音频。
3. **右侧磁吸弹窗**（默认开）：从屏幕右侧滑入的常驻小窗口：
   - 按博主合并未读卡片（同名多条显示数字角标），点击打开原帖并标记该博主已读
   - **「全部已读」按钮**：一键把所有未读标记为已读，卡片全滑出后自动关窗
   - 全读完自动关闭；3 分钟未读会重新弹出
4. **企微推送到微信**（可选，见下方④）：同时发到你的个人微信——**电脑前桌面微信跳出、离开电脑手机微信收到**

### 使用场景

| 场景 | 你看到什么 | 怎么最醒目 |
|---|---|---|
| 在电脑前工作 | 桌面弹窗从右侧磁吸入 + 声音 + 系统通知 | 弹窗常驻，点卡片或「全部已读」处理 |
| 电脑开着但不在屏幕前 | 桌面微信跳出企微消息 | 微信置顶该会话 + 关免打扰 |
| 手机上（离开电脑） | 手机微信收到企微消息 | 同上：置顶会话 = 最接近"弹窗"的原生形态 |
| 雪球原生推送（备用） | 手机系统通知 | 可同时开，但会淹没在全量通知里，不如企微过滤后的醒目 |

> **手机端实操建议**：在微信里把企微推送的接收会话**置顶 + 关闭免打扰**——以后特别关注一发帖，微信列表最顶部直接弹出、带红点角标。这是手机上不用装 App 就能达到的最醒目效果。

## ④ 企业微信推送（把新帖发到你的个人微信）

扩展支持把每条新帖**额外**推送到你的个人微信，走企业微信自建应用（免费、不限量）。配置一次后，雪球新帖会同时触发：本机通知 + 声音 + 弹窗 + 微信消息。

### 前置：在企业微信后台建一个应用（个人也能建，无需公司/营业执照）

> 企业微信 ≠ 必须有公司。用你的**个人微信扫码**就能创建一个「企业」（名字随意，如「我的提醒」），创建后你就是管理员，能随意建应用、拿凭证。**这个应用纯粹是个「消息出口」，不用审核、不用上架。**

1. **进后台**：打开 [企业微信管理后台](https://work.weixin.qq.com)，用**微信扫码**登录。若提示「创建企业」，按引导建一个即可（企业名称随便填，类型选「个人 / 团队」，**无需认证**也能用推送功能）。
2. **拿 CorpID**：左侧「我的企业」→「企业信息」→ 拉到页面底部，复制 **CorpID（也叫企业 ID）**。
3. **建应用**：左侧「应用管理」→「自建」→「创建应用」→ 上传个 logo（随便选）、起名（如「雪球推送」）、「可见范围」选**你自己** → 创建。进入应用后复制 **AgentID**；点 **Secret 旁的「查看」**（会要求你用微信扫码或短信验证一次）→ 复制 **Secret**。
4. **⚠️ 关键一步：开启微信接收**：在这个应用详情页里找到「在微信中接收 / 在微信插件中接收」，把它**开启**。这一步决定消息进不进你的**个人微信**——不开的话消息只进企业微信 App，你的微信里收不到。
5. （可选）在手机微信里确认已能收到该企业微信应用的消息（微信「通讯录」→「企业微信联系人」里能看到它）。

### 在扩展里填写

打开扩展**设置页 → ④ 企业微信推送**：

- 勾选「启用企业微信推送」
- 填入 CorpID / Secret / AgentID（就是上面拿到的三个值）
- 接收人 UserID：**直接留空即可**（= 发给应用可见范围内的所有人，也就是你自己，省去查 UserID 的麻烦）；若要指定，可在企微后台「通讯录」里查自己的 UserID，多人用 `|` 分隔
- 点「📤 测试企微推送」—— 若微信立刻收到一条测试消息，说明配置成功；否则按错误提示修正

> ⚠️ Secret 属于敏感凭证，仅保存在你本机浏览器（`chrome.storage.local`），不会上传任何服务器。

### 工作原理

新帖触发时，`background.js` 的 `pushWecom()` 会：① 用 CorpID+Secret 换 `access_token`（缓存 2 小时，提前 5 分钟复用）；② 调 `message/send` 把消息发到指定 UserID。整个过程走 `https://qyapi.weixin.qq.com`，manifest 已声明对应主机权限。

## 已知限制

- ✅ 本机屏幕醒目弹窗（磁吸常驻 + 全部已读）—— 已满足。
- ✅ 推到个人微信（桌面跳出 + 手机收到，只过滤特别关注、不淹没）—— 可选。在设置页「④ 企业微信推送」配置后生效。
- ⚠️ 依赖 Chrome 常开：扩展无法在 Chrome 完全关闭后运行（这是 MV3 的硬限制）。若想要"Chrome 关了也推"，可加 GitHub Actions 定时 workflow 调企微 API（需定期更新雪球 Cookie 到 Secrets），当前版本暂不包含此功能。
- Chrome 必须保持打开且雪球已登录，否则收不到提醒。

## 故障排查：一键诊断

如果「检测登录」一直失败 / 获取不到，打开设置页**最下方**点 **「🔧 一键诊断」**：

- 它会逐条测试：① 是否有 xueqiu.com 标签页 → ② 每个标签页的 content script 是否注入并响应（次路径，未注入可忽略）→ ②b 直接用 `scripting` 注入主路径是否可用 → ③ `chrome.cookies` 是否可用 → ④ 是否手动粘贴了 Cookie → ⑤ 真实跑一次 `friendships/groups.json`（同时兼做登录探针，验证主路径端到端成功）。
- 每一步显示 ✅/❌ 及关键信息（标签页状态、返回内容预览、错误原因、修复建议）。
- 点「📋 复制报告」，把内容发给我即可精确定位问题。

> 注意：雪球原 `user/current.json` 登录校验接口已失效（返回 404），现改用 `friendships/groups.json` 作为登录探针——未登录返回 `error_code:400016`，已登录返回分组列表。

常见卡点与对策：

| 现象 | 含义 | 解决 |
|---|---|---|
| ① 标签页 count=0 | Chrome 没开 / 没登录雪球 | 打开 xueqiu.com 并登录 |
| ② ❌ Content script 未注入 | 雪球页面没刷新、扩展脚本没装上（次路径缺失） | **可忽略**：只要 ②b ✅，主路径正常，无需 F5 |
| ② ✅ 但 verdict=返回未登录 | 雪球仍认为未登录 | 看 preview 字段具体内容，发我分析 |
| ② ❌ 但 ②b ✅ | 仅 content script 没注入，主路径可用 | 正常，**无需任何操作** |
| ② ❌ 且 ②b ❌ | 主路径拿不到数据（页面未开 / 未登录 / 反爬拦截） | 打开并登录 xueqiu.com；必要时手动粘贴 Cookie |
| ⑤ ❌ 无法获取登录态 | 三条路径全失败 | 按提示刷新页面，或在「高级」里手动粘贴 Cookie |

### 「有人发帖了但没提醒」怎么查

1. 先点设置页「📢 发送测试提醒」：若右下角通知 + 置顶小窗**正常弹出**，说明通知/弹窗链路没问题，问题在抓取或比对环节。
2. 点扩展图标看面板：
   - **监控博主：N 人** —— 若 N=0，说明没拉到「特别关注」分组（检查登录态、分组是否存在）。
   - **抓取: 张三(5) 李四(3)** —— 括号数字是该博主本次拉到的帖数；若某人显示 `李四:错误`，说明他时间线拉取出错（多为登录态失效）。
   - **⚠️ 上次检查出错** 标红 —— 按红字内容对症处理。
3. 仍查不出 → 设置页「🔧 一键诊断」发报告。

### 📜 运行日志（出问题时发我分析）

设置页底部新增「📜 运行日志」面板。扩展在后台会**自动**记录每一步：

- 轮询触发 / 安装 / 浏览器启动
- 取数走了哪条路径（① scripting 注入 / ② content script / ③ Cookie 注入）及是否成功
- 每用户时间线抓取条数 / 失败原因
- 弹窗定位坐标（left/top + 所用策略）
- 每条 ERROR / WARN 及堆栈

日志持久化在 `chrome.storage.local`，service worker 重启后自动读回，不丢失：
- INFO / WARN 滚动保留最近 300 条（高频轮询也不无限增长）；
- **ERROR 永久保留，不进 300 条滚动上限、不会被轮询冲刷**；点「📋 复制日志」后即清空（已交出去无需再留）。

面板提供：**🔄 刷新 / 📋 复制 / 💾 下载 .txt / 🗑 清空**。

当提醒异常又查不出原因时，直接点「📋 复制日志」或「💾 下载」把内容发我，即可精确定位。
